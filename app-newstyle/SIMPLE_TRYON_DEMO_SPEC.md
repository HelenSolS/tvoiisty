SIMPLE_TRYON_DEMO_SPEC.md
Лёгкий демонстрационный интерфейс Try-On
Цель

Сделать очень простой демонстрационный интерфейс, который использует уже существующий backend и AI-pipeline.

Это не часть основного приложения.
Это демо-страница для инвестора, которая показывает работу технологии.

Сценарий максимально простой:

Фото человека → Фото одежды → Примерить → Результат
Важно

Это демо поверх существующего backend.

Backend уже делает всё:

принимает изображения

запускает AI

возвращает результат примерки

Фронт должен только:

1 загрузить фото человека
2 загрузить фото одежды
3 нажать "Примерить"
4 показать результат
Главное правило

Ничего из существующего проекта переписывать нельзя.

Нельзя:

менять backend

менять API

менять очередь

менять workers

менять try-on pipeline

менять текущий интерфейс

менять state приложения

менять галерею

менять историю

менять мои фото

Это отдельная демо-страница, которая просто вызывает API.

Где разместить

Создать отдельную страницу:

/demo/simple-tryon

или отдельный файл:

simple-tryon.tsx

Страница должна быть изолирована от основного интерфейса.

Она не должна импортировать сложные компоненты приложения.

Интерфейс

Интерфейс должен быть очень простой.

3 шага.

Блок 1 — загрузка фото человека
Загрузите фото человека
[Выбрать файл]

После выбора файл отображается в карточке.

Блок 2 — загрузка фото одежды
Загрузите фото одежды
[Выбрать файл]

После выбора файл отображается в карточке.

Блок 3 — кнопка примерки
[ПРИМЕРИТЬ]

Кнопка активна только если загружены оба изображения.

Результат

После нажатия:

ПРИМЕРИТЬ

появляется блок:

Результат примерки

и отображается изображение результата.

API которые нужно использовать

Использовать только существующие endpoints.

1 загрузка изображения
POST /api/media/upload

Отправляем файл.

Получаем ответ:

{
  id
  url
}
2 запуск примерки

После загрузки двух изображений вызываем:

POST /api/tryon

Отправляем:

photoId
lookId

или если используется URL:

personImage
garmentImage

Использовать тот же контракт, который используется в основном приложении.

Что не использовать

На демо-странице не должно быть:

Галереи

Истории

Моих фото

garmentMemory

backendLooks

сложного state

localStorage

авторизации

меню

режимов просмотра

слайдеров

сложной логики

Это чистый минимальный интерфейс.

Поведение

Алгоритм работы страницы:

1 пользователь загружает фото человека
2 пользователь загружает фото одежды
3 фронт отправляет оба файла в /api/media/upload
4 получает два id
5 нажимает ПРИМЕРИТЬ
6 фронт вызывает /api/tryon
7 backend запускает AI
8 возвращается результат
9 результат отображается

Backend уже делает всю работу.

Фронт — только интерфейс.

Дизайн

Дизайн должен быть:

минималистичный

аккуратный

стильный

лёгкий

Использовать канон дизайна проекта.

Важно:

светлый интерфейс

спокойные стильные тона

крупные карточки

аккуратные кнопки

много воздуха

Должно выглядеть как дорогой продукт, но без сложной логики.

Визуальная структура

Пример layout:

Заголовок
AI Try-On Demo

[Карточка 1]
Загрузите фото человека

[Карточка 2]
Загрузите фото одежды

[Кнопка]
ПРИМЕРИТЬ

↓

Состояния
ожидание загрузки
идёт обработка AI
результат готов

Во время обработки показывать:

AI обрабатывает изображение
Очень важно

Этот интерфейс — игрушка поверх настоящего backend.

Но backend при этом:

реально принимает изображения

реально запускает AI

реально делает примерку

То есть:

фронт простой
backend настоящий
AI настоящий
Проверка

Перед PR нужно проверить:

1 загрузить фото человека
2 загрузить фото одежды
3 нажать примерить
4 получить результат

Проверить несколько раз.

Ограничения

Этот интерфейс:

не должен ломать основное приложение

не должен менять существующие файлы

не должен влиять на состояние приложения

Он существует только как демо-страница.

Итог

Получается простой сценарий:

фото 1
фото 2
кнопка
AI делает примерку
инвестор счастлив

Backend уже умеет всё остальное. и Как выглядит UI, который обычно “заходит” инвесторам

Страница почти пустая. В центре один понятный блок.

AI Try-On

[ фото человека ]   +   [ фото одежды ]

          ↓

        TRY ON

          ↓

       Результат

Почему это работает:

видно всю логику сразу

ничего не отвлекает

создаёт ощущение дорогого продукта

Что я бы поправил в макете

1️⃣ Сделать два блока одинаковыми

Person
Clothing

2️⃣ Между ними большой знак +

Это визуально объясняет:

человек + одежда = результат

3️⃣ Кнопка TRY ON одна большая

4️⃣ Результат появляется ниже на той же странице

Цвета, которые обычно нравятся инвесторам

Очень спокойная палитра:

Фон:

#f6f7fb

Карточки:

#ffffff

Акцент кнопки:

#5b6cff

Текст:

#0f172a

Это выглядит дорого и технологично.

Почему такой интерфейс лучше для демо

Инвестор не будет разбираться в:

галерее

истории

режимах

меню

Он смотрит 10 секунд и должен понять:

Ага.
Фото → одежда → AI → результат.
Важный психологический момент

Если интерфейс слишком сложный, инвестор начинает думать:

система сырая

Если интерфейс очень простой, он думает:

технология готова
