# 📌 VT-MAGIC 2026 - Техническая спецификация (ТЗ)

## 0. Видение проекта
Минималистичная, люксовая виртуальная примерочная. Фокус: 2026 год.

## 1. Карта функций и текущий статус

### 1.1 Ролевая модель доступа (RBAC)
- **USER (Пользователь)**: 5 примерок в день. Нет доступа к панели Магии. Роль сохраняется в браузере.
- **MAGIC_USER (Премиум)**: 100 примерок в день + 10 видео "Магия" в день. Доступ к панели Магии и Студии контента.
- **SHOP_USER (Магазин)**: Функции примерки + Управление коллекциями 2026.
- **ADMIN (Админ)**: Безлимитные генерации. Переключатель AI-провайдеров. Управление пресетами.

### 1.2 Виртуальная примерка (VTO)
- **Логика**: Фото пользователя + Фото одежды = Результат.
- **Провайдеры**: 
  - **Gemini**: `gemini-3-pro-image-preview`
  - **Custom**: Резервный API для Imagen.
- **Сезон**: Весна-Лето 2026.

### 1.3 Панель Магии (Видео)
- **Логика**: Работает только на основе **Результата** примерки.
- **Модель**: `veo-3.1-fast-generate-preview`
- **Статус**: Включено для ролей Magic и Admin.

### 1.4 Студия контента (Content Factory)
- **Генератор подписей**: Gemini 3 Flash создает 3 варианта текста для поста.
- **Шеринг**: Прямой экспорт и публикация в соцсети (IG, TikTok, TG).
- **Водяные знаки**: Автоматический брендинг результата.

### 1.5 Магазины и Коллекции
- **Год**: Коллекции 2026.
- **Поток**: Пользователь просматривает бутики, выбирает коллекцию, выбирает вещь для примерки.
- **Ссылки**: Вещи ведут на внешние сайты магазинов.

## 2. Технические детали реализации
- **Постоянство данных**: `localStorage` хранит роль, тему и ежедневные лимиты.
- **Ошибки**: Сбой кастомного сервиса переключает систему на Gemini.
- **API Ключи**: Для моделей Veo и Gemini 3 Pro используется выбор ключа через `aistudio` интерфейс.
