# Деплой и аварийное восстановление сервера

Скрипты рассчитаны на запуск **на сервере** из корня проекта (например `/opt/newstyle/newstyle`).

## Стандартный деплой

Обновляет репозиторий (ветка `dev`), пересобирает и запускает контейнеры, проверяет health.

```bash
./scripts/deploy.sh
```

С указанием ветки:

```bash
./scripts/deploy.sh main
```

Через Makefile:

```bash
make deploy
```

## Аварийное восстановление

Используется при зависании docker-proxy, конфликтах портов, нерабочем docker.socket. Останавливает контейнеры и docker-сервисы, завершает процессы docker-proxy, перезапускает Docker, чистит сети и поднимает стек заново.

**Требуются права на `systemctl` (обычно root).**

```bash
./scripts/recover.sh
```

Через Makefile:

```bash
make recover
```

## Архитектура (docker-compose)

- **API** — единственный сервис с публичным портом: `4000:4000`.
- **Postgres** и **Redis** — без публикации портов, доступ только из docker network (устраняет зависания docker-proxy и конфликты портов).
- **Worker** — без портов.

## Health check

После деплоя и восстановления проверяется `http://localhost:4000/health`. Убедитесь, что nginx (если используется) проксирует на порт 4000.
