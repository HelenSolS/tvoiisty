# MVP Architecture — Примерочная

## 1. Общая цель

Система **«Примерочная»** — мобильный веб-интерфейс для примерки готовых образов на фото пользователя.

- **Интерфейс** уже существует и утверждён инвестором.
- **Backend** должен быть реализован заново, но строго поддерживать этот интерфейс.

Система должна:

- быстро открываться
- работать на мобильных устройствах
- поддерживать до ~1000 пользователей в день
- иметь устойчивый pipeline генерации изображений

---

## 2. Основные экраны приложения

Интерфейс состоит из трёх основных экранов.

### Мои фото

Галерея пользовательских фотографий.

**Функции:**

- загрузить фото
- выбрать фото для примерки
- удалить фото
- максимум 10 фото

**Особенности:**

- по умолчанию используется последнее загруженное фото

### Образы

Галерея образов. Образы добавляются администраторами.

**Функции:**

- листать образы
- лайкнуть
- примерить

### История примерок

Содержит результаты генераций.

**Функции:**

- открыть результат
- скачать изображение
- поделиться
- анимировать
- открыть видео

---

## 3. Поведение галерей

В каждом экране используется единая механика.

- **режим иконок** — grid из карточек
- **режим просмотра** — карточка раскрывается во весь экран

**Поведение:**

- карточка плавно увеличивается
- пользователь может свайпать
- при закрытии карточка возвращается в grid

---

## 4. Архитектура системы

```
frontend
   │
   ▼
backend API
   │
   ├ PostgreSQL
   ├ Redis (queues)
   ├ Supabase Storage
   │
   ├ Fal (primary try-on)
   └ KIE (fallback + video)
```

---

## 5. Основные компоненты

### frontend

Мобильный интерфейс.

**Особенности:**

- 9:16
- большие изображения
- минималистичный UI
- никаких прямых вызовов AI

### backend

Node.js сервис.

Отвечает за:

- API
- очереди
- хранение медиа
- pipeline генераций

### PostgreSQL

Основная база данных.

Хранит:

- пользователей
- образы
- медиа
- историю генераций

### Redis

Используется для очередей задач.

### Supabase Storage

Используется для хранения:

- изображений
- видео
- образов

---

## 6. Хранение изображений

### пользовательские фото

**Ограничения:**

- максимум 10
- TTL 15 дней

**Поведение:**

- при загрузке 11-го фото удаляется первое
- если фото удалено с сервера, пользователь может загрузить его заново

### образы

Хранятся постоянно.

### результаты примерки

Сохраняются обязательно.

### видео

Сохраняется только последнее видео генерации.

---

## 7. Pipeline примерки

```
client
   ↓
POST /tryon
   ↓
create tryon_session
   ↓
enqueue job
   ↓
worker
   ↓
Fal try-on
   ↓
retry
   ↓
KIE fallback
   ↓
save result
   ↓
update session
```

---

## 8. Таймауты

Fal генерации могут занимать до 60 секунд.

Система должна поддерживать:

- **server timeout:** 70 сек
- **worker timeout:** 65 сек

---

## 9. Масштабирование

- **начальная нагрузка:** 100 пользователей / день
- **ожидаемая:** 1000 пользователей / день
- инфраструктура одного сервера достаточна
