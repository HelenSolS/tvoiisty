# MEDIA_PIPELINE

## 1. Общий принцип

Все изображения должны проходить через media pipeline.

Нельзя напрямую отправлять изображения в AI.

Pipeline обеспечивает:

- проверку
- сжатие
- безопасное хранение
- стабильные ссылки.

## 2. Источники изображений

В системе существует три типа изображений.

- **пользовательские фото** — загружаются пользователем.
- **изображения образов** — загружаются администраторами.
- **результаты генерации** — создаются AI.

## 3. Сжатие и нормализация изображений

Перед отправкой в AI изображения должны быть сжаты.

Причины:

- ограничение размера API
- ускорение генерации
- уменьшение ошибок.

**Загрузка пользовательского фото (Stage 0):**

- **Нормализация:** максимальная сторона **2048 px**, формат JPEG, качество 80.
- **Превью:** максимальная сторона **512 px**, JPEG, качество 80 (отдельный файл в storage).

**Ответ API загрузки:** `{ id, url, previewUrl }` — id актива в media_assets, url основной картинки, previewUrl превью.

Для отправки в AI далее может применяться дополнительное сжатие (например 1536 px).

## 4. Хранение

Все изображения хранятся в Supabase Storage.

Структура bucket:

```
media/
   users/
       {userId}/
           photos/
           results/
           videos/

   looks/
```

## 5. Временные изображения

Пользовательские фото хранятся временно.

**TTL:** 15 дней

## 6. Ограничение количества фото

**Максимум:** 10

При загрузке 11 фото:

удаляется первое.

## 7. Результаты генерации

Результаты должны сохраняться сразу после генерации.

Нельзя хранить только ссылки AI-сервисов.

## 8. Видео

Видео сохраняется отдельно.

Система хранит только последнее видео генерации.
